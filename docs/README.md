# SNOWICK MASTER SYSTEM

Structure-Engineered Brand Operation System

---

## 프로젝트 목적

스노윅 운영 구조를 GitHub 기준 문서로 통합 관리한다.

관리 대상:

- SEO
- 상세페이지
- 라벨
- 자동화
- 마켓 운영

GitHub는 단순 저장소가 아니라:

- 스노윅 AI 규칙 DB
- 구조 기준 원장
- 자동화 기준 시스템
- 브랜드 운영 메모리

역할로 사용한다.

---

## 핵심 운영 방향

- 구조 우선
- 재현성 유지
- 단일 실행 경로 유지
- 재작업 방지
- 플랫폼 구조 분리
- 감성보다 구조 우선

---

## 시스템 구성

---

## 운영 원칙

1. 파일명은 반드시 영문으로 유지한다.
2. snake 및 underscore 구조를 유지한다.
3. 내부 문서 내용은 한글 사용 가능하다.
4. 구조 변경보다 내용 확장을 우선한다.
5. 중복 시스템 파일 생성을 금지한다.
6. 기존 파일명 재수정을 금지한다.
7. GitHub 모바일 표시 오류보다 실제 편집 데이터를 우선 판단한다.

AUTOMATION_SYSTEM.md

---

## 수정 원칙

- 새 파일 생성보다 기존 문서 확장을 우선한다.
- v2, final, real, last 형식의 파일 생성을 금지한다.
- 검증된 운영 규칙만 누적 기록한다.
- 임시 테스트 내용을 원장에 기록하지 않는다.
- 구조 변경은 사전 검토 후 진행한다.

---

## 스노윅 운영 방향

스노윅은 감성이 아니라 구조를 설계한다.

운영 기준은:

- 안정성
- 재현성
- 구조 일관성
- 장기 유지 가능성

을 우선한다.

## 현재 운영 상태

SNOWICK_GITHUB_MASTER_SYSTEM_V1 구조 기준으로 운영한다.
