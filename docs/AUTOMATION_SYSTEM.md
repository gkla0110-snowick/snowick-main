# 스노윅 자동화 시스템

---

## 핵심 방향

- 구조 우선
- 단일 실행 경로 유지
- 재현 가능한 워크플로우 유지
- 롤백 가능한 구조 유지
- 중복 프로세스 생성 금지
- 자동화보다 안정성 우선

---

## 자동화 운영 목적

스노윅 자동화는:
단순 속도 향상이 목적이 아니다.

자동화는:

- 반복 작업 감소
- 구조 안정성 유지
- 운영 기준 일관성 유지
- 재작업 방지
- 오류 감소

를 목적으로 운영한다.

## 롤백 기준

다음 상황 발생 시:
자동화를 즉시 중단한다.

- 중복 발송
- 데이터 꼬임
- 상태값 충돌
- 플랫폼 정책 변경
- API 오류 반복
- 예외 처리 실패
- 구조 충돌 발생

중단 후:
구조 점검을 우선 진행한다.

## 구조 유지 기준

- 단일 실행 경로 유지
- 반복 가능한 구조 유지
- 예외 처리 가능 구조 유지
- 운영 로그 유지
- 구조 변경 최소화

## 금지 사항

- 테스트 없이 실운영 진행
- 중복 자동화 생성
- 수동 작업과 자동화 혼합 운영
- 예외 처리 없는 배포
- 로그 없는 자동화 운영
- 구조 미확정 상태 자동화 구축
- 임시 자동화 장기 운영

## 스노윅 자동화 방향

스노윅 자동화는:
속도보다 구조 안정성을 우선한다.

자동화는:
사람을 대체하는 목적이 아니라

- 반복 작업 감소
- 운영 기준 유지
- 재현성 확보
- 오류 감소

를 목적으로 운영한다.
