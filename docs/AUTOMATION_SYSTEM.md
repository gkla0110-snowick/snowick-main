# 스노윅 자동화 시스템

## 핵심 방향

- 구조 우선
- 단일 실행 경로 유지
- 재현 가능한 워크플로우
- 롤백 가능한 구조
- 중복 프로세스 금지
- 자동화보다 안정성 우선

---

## 자동화 목적

자동화는:
작업 속도를 높이기 위한 것이 아니라

- 반복 작업 감소
- 구조 안정화
- 재작업 방지
- 운영 일관성 유지

를 목적으로 한다.

---

## 핵심 시스템

- Gmail 자동 분류
- Notion 데이터 관리
- AI 고객 응답
- 상품 정보 관리
- 마켓 운영 보조
- SEO 작성 보조
- 상세페이지 작성 보조

---

## 운영 원칙

- 구조 확정 전 자동화 금지
- 테스트 없이 실운영 금지
- 중복 시나리오 생성 금지
- 동일 기능 자동화 반복 금지
- 임시 자동화 장기 사용 금지

---

## Make 시스템 규칙

- 단일 시나리오 우선
- 모듈 역할 분리
- 중복 라우터 최소화
- 오류 발생 시 즉시 중단 가능 구조 유지
- 로그 확인 가능 상태 유지

---

## Gmail 규칙

- unread 기준 사용
- 자동 회신 중복 방지
- noreply 필터 우선 제외
- 수동 검토 단계 유지
- AI_REVIEW 구조 유지

---

## Notion 규칙

- 중복 데이터 생성 금지
- 상태값 기준 운영
- 검색 후 생성 구조 유지
- 임시 DB 남발 금지
- 단일 기준 DB 유지

---

## AI 응답 규칙

- JSON 구조 우선
- 구조 검증 후 발송
- 제품 DB 기준 응답
- FAQ 기준 우선
- 불확실 응답 자동 제한

---

## 롤백 기준

다음 상황 시 자동화 중단:

- 중복 발송
- 데이터 꼬임
- 상태값 충돌
- 플랫폼 정책 변경
- API 오류 반복
- 예외 처리 실패

즉시 구조 점검 후 재시작한다.

---

## 금지 사항

- 테스트 없이 실운영
- 중복 자동화 생성
- 수동 작업과 자동화 혼합 운영
- 예외 처리 없는 배포
- 로그 없는 자동화 운영
- 구조 미확정 상태 자동화 구축

---

## 스노윅 자동화 방향

스노윅 자동화는:
속도보다 구조 안정성을 우선한다.

자동화는:
사람을 대체하는 것이 아니라

- 반복 작업 감소
- 운영 기준 유지
- 재현성 확보
- 오류 감소

를 목표로 운영한다.
