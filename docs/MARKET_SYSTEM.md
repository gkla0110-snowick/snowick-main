# SNOWICK MARKET SYSTEM

## CORE RULE

- Cafe24 centered
- Single source management
- No duplicated product editing
- Stable synchronization priority
- Marketplace separation maintained

---

## MAIN SOURCE

Cafe24 = Master product DB

All marketplace data must originate from Cafe24.

Direct marketplace editing prohibited.

---

## MARKET FLOW

Cafe24
→ MarketPlus
→ SmartStore
→ Coupang
→ KakaoTalk Store

---

## SMARTSTORE RULE

- SEO optimized title
- Mobile readability priority
- Promotion text under 50 bytes
- Search optimized tags
- Thumbnail optimization

---

## COUPANG RULE

- Search optimized keywords
- Category optimization
- Exposure focused structure
- Duplicate keyword minimization

---

## KAKAO RULE

- Product direct editing prohibited
- MarketPlus linkage priority
- Single source synchronization
- Product DB consistency maintained

---

## PRODUCT RULE

Each product must maintain:
- Unified naming
- Unified SEO
- Unified pricing
- Unified image structure
- Unified specification

---

## IMAGE RULE

- White tone priority
- Real product centered
- Platform optimized thumbnail
- Mobile optimized readability

---

## SAFETY RULE

Prohibited:
- Direct marketplace modification
- Duplicated uploads
- Broken synchronization
- Mixed product DB structure

---

## SYSTEM GOAL

Maintain unified Snowick marketplace management structure.

# 스노윅 마켓 시스템

## 핵심 방향

- 카페24 단일 원장 유지
- 플랫폼별 구조 분리
- 마켓플러스 중심 운영
- 중복 매핑 방지
- 재작업 방지 구조 유지
- 직접 수정 최소화

---

## 운영 기준

카페24를 기준 원장으로 사용한다.

모든 상품 수정은:
카페24 기준으로 진행한다.

플랫폼 직접 수정은 최소화한다.

---

## 플랫폼 구조

- 카페24
- 마켓플러스
- 카카오톡스토어
- 네이버 스마트스토어
- 쿠팡

각 플랫폼은:
동일 구조가 아니라
플랫폼 특성 기준으로 분리 운영한다.

---

## 카카오 운영 규칙

- 카카오 직접 수정 금지
- 카페24 → 마켓플러스 → 카카오 구조 유지
- 부분 수정 반복 금지
- 상품 DB 꼬임 발생 시 부분 복구 금지
- 구조 재정비 우선

---

## 마켓플러스 규칙

- 상품 매핑 중복 금지
- 동일 상품 중복 생성 금지
- 플랫폼별 전송 상태 확인
- 테스트 상품 먼저 검증
- 정상 확인 후 전체 전송

---

## 스마트스토어 규칙

- 플랫폼 정책 우선 준수
- 자동완성 기반 SEO 유지
- 제한 키워드 검증
- 플랫폼 전용 문장 사용

---

## 쿠팡 규칙

- 플랫폼 검색 구조 우선
- 태그 구조 분리
- 제품 용도 중심 구성
- 중복 키워드 금지

---

## 금지 사항

- 플랫폼 직접 수정 남발
- 중복 상품 생성
- 테스트 없이 전체 전송
- 카카오 수동 수정 반복
- 플랫폼 문장 동일 복붙
- 구조 변경 후 즉시 대량 작업

---

## 문제 발생 시 기준

우선 판단:

1. 구조 문제인지
2. 매핑 문제인지
3. 플랫폼 정책 문제인지
4. 동기화 문제인지

순서대로 판별한다.

즉시 수정 반복보다:
원인 구조 파악을 우선한다.

---

## 스노윅 마켓 방향

스노윅은:
속도보다 구조 안정성을 우선한다.

운영 기준은:
- 단일 원장
- 재현 가능 구조
- 플랫폼 분리
- 재작업 방지

를 중심으로 유지한다.
