시스템 목적
────────────────────────

스노윅 마켓 운영 구조를 단일 원장 기준으로 통합 관리한다.

마켓 운영은 단순 상품 업로드가 아니라 다음 목적을 위해 운영한다.

1. 구조 안정성 유지
2. 플랫폼 분리 운영
3. 데이터 일관성 유지
4. 반복 가능한 운영 유지
5. 중복 매핑 방지
6. 재작업 방지
7. 운영 원장 기준 유지

────────────────────────
카페24 운영 기준
────────────────────────

카페24를 기준 원장으로 사용한다.

운영 기준:

1. 모든 상품 수정은 카페24 기준으로 진행한다.
2. 상품명, 요약, 간략설명, 추가설명, SEO, 이미지 ALT 기준을 카페24에 우선 반영한다.
3. 플랫폼별 전송 전 카페24 원장 상태를 먼저 확인한다.
4. 마켓 직접 수정은 최소화한다.
5. 카페24와 마켓 상품 정보가 충돌하면 카페24 원장을 우선한다.

카카오 운영 기준
────────────────────────

운영 기준:

1. 카카오 직접 수정 금지
2. 카페24 → 마켓플러스 → 카카오 구조 유지
3. 부분 수정 반복 금지
4. 상품 DB 꼬임 발생 시 부분 복구 금지
5. 구조 재정비 우선
6. 계정 혼합 운영 금지
7. 기존 꼬인 상품 억지 복구 금지

카카오톡스토어 DB 꼬임 판정 기준:

1. 카카오톡스토어에는 상품이 존재하지만 마켓플러스 상품가져오기 DB에 정상 노출되지 않음
2. 신규 전송 시 중복상품 오류 발생
3. 과거 허브애향 계정 또는 다른 카카오 계정 연동 이력 존재
4. 현재 스노윅 계정 기준 재연동 중
5. API 인증은 정상이나 상품 DB 매핑이 불안정함

DB 꼬임 판정 시 금지:

1. 부분 수정 금지
2. 재인증 반복 금지
3. 기존 상품 억지 복구 금지
4. 카카오 판매자센터 직접 수정 금지
5. 계정 변경 반복 금지

권장 단일 실행 경로:

1. 카카오톡스토어 기존 상품 전체 삭제
2. 카페24 상품 DB 기준 원장 확정
3. 마켓플러스 카카오톡스토어 연동 유지
4. 카페24 상품을 마켓플러스 일괄보내기로 재전송
5. 카카오톡스토어 카테고리 연결
6. 채널홈 최신상품·상품검색·배너 검증
7. 이후 카카오 판매자센터 직접 상품 수정 금지
