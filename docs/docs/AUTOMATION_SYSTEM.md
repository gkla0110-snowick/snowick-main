2. 운영 기준 유지
3. 구조 안정성 유지
4. 오류 감소
5. 재작업 방지
6. 중복 처리 방지
7. 실운영 전 롤백 가능성 확보

2. Notion 운영 원장 확정
3. Make 실행 흐름 설계
4. Gmail 수신 감지 연결
5. Notion 중복 검색 구조 생성

7. AI 문의 분류
8. AI 답변 초안 생성

10. 자동 회신 검토
11. 실운영 전환

현재 기준 단일 실행 경로:

Notion
→ Make
→ Gmail

이 순서가 맞는 이유:

1. Notion은 운영 원장이다.
2. Make는 실행 엔진이다.
3. Gmail은 데이터 수신 출발점이다.
4. Notion 구조가 확정되지 않으면 Make 시나리오 재작업이 발생한다.
5. Gmail부터 연결하면 데이터 저장 기준이 흔들린다.

────────────────────────

Gmail 연결 기준:

snowick@naver.com 기준으로 진행한다.

Gmail Watch Emails 기준:

1. Only unread 기준 사용
2. 읽지 않은 메일만 감지
3. 처리 완료 후 라벨 또는 읽음 처리 기준을 별도 적용
4. 자동 회신은 초기 단계에서 사용하지 않음

제외 대상:

1. noreply
2. no-reply
3. mailer-daemon
4. bounce
5. newsletter
6. 광고성 메일
7. 시스템 알림
8. 결제 시스템 자동 알림
9. 배송사 자동 알림
10. 스팸성 반복 메일

Gmail 운영 라벨:

1. AI_REVIEW
2. AI_DONE
3. AI_ERROR
4. AI_DUPLICATE

라벨 기준:

AI_REVIEW:
수동 검토가 필요한 메일에 적용한다.

AI_DONE:
처리 완료된 메일에 적용한다.

AI_ERROR:
JSON 오류, Notion 저장 오류, 분류 실패 등 자동화 오류 발생 시 적용한다.

AI_DUPLICATE:
원본메일ID 기준으로 이미 처리된 메일에 적용한다.

금지 기준:

1. 중복 회신 금지
2. 동일 thread_id 반복 회신 금지
3. 원본메일ID 없는 상태에서 자동 저장 금지
4. 수동 검토 대상 자동 발송 금지
5. Gmail 자동 Reply는 최종 승인 전 사용 금지
