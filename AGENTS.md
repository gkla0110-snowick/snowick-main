# SNOWICK Codex Operating Rules

## File Role

This file is the primary instruction file for Codex in the SNOWICK GitHub repository.

Codex must read this file before editing, creating, reviewing, or proposing any change.

## Repository Identity

Repository purpose:

```text
SNOWICK MASTER SYSTEM
Structure-Engineered Brand Operation System
```

SNOWICK is not a general lifestyle fragrance brand.
SNOWICK is a structure-engineered fragrance and combustion system brand.

Priority order:

```text
structure > reproducibility > safety > platform accuracy > expression
```

Before working, Codex must read the available files in this order:

If a listed file does not exist, do not invent its contents.
Continue only with existing files and state which expected file was missing in the pull request summary.

## Operating Boundary

Codex may:

- edit Markdown files
- generate Markdown files
- update SEO text
- update product descriptions
- update SNS captions
- update label copy
- update Cafe24 field drafts
- update Coupang search terms
- update Naver Shopping search term documents
- clean up Markdown structure
- create branches
- create pull requests
- add pull request comments
- propose corrections

Codex must not:

- merge pull requests
- delete files
- create repositories
- fork repositories
- publish to Cafe24
- publish to Coupang
- publish to Naver Smart Store
- publish to Instagram
- publish to TikTok
- publish to YouTube
- change confirmed prices
- change confirmed product names
- change confirmed product specifications
- invent patent numbers
- invent trademark claims
- invent legal claims
- invent performance claims
- invent verified search terms
- overwrite unrelated content

## Git Workflow

For every task:

```text
1. Read required files.
2. Identify target file.
3. Modify only the requested section.
4. Preserve existing structure.
5. Do not delete content unless the user explicitly requested removal.
6. Create a new branch.
7. Create a pull request.
8. In the PR summary, include:
   - changed files
   - changed sections
   - reason for change
   - validation checklist
   - items requiring human approval
```

Do not merge the pull request.

## Human Approval Rule

Codex is allowed to propose.
Codex is not allowed to finalize business decisions.

Final authority belongs to the SNOWICK representative.

Codex must mark the following as requiring human approval:

- price changes
- product name changes
- legal or IP statement changes
- platform publishing
- search terms not yet confirmed by platform input
- claims related to safety, performance, or certification
- new product structure
- deletion or archival of files

## Product Description Rule

When writing product descriptions, follow this order:

```text
1. 소비자 문제
2. 스노윅 구조 차이
3. 제품 근거
4. 사용 환경
5. 구매 이유
6. 행동 유도
```

Every product description must answer:

```text
왜 일반 제품이 아니라 스노윅인가
```

Do not start with vague emotional language.
Start from the consumer problem or structural difference.

## Naver Shopping Rule

Naver Shopping search terms must be separated into confirmed terms and candidate terms.

Confirmed registerable terms:

```text
소이캔들
생일선물
우드심지
집들이선물
```

Candidate terms are not final terms until the user confirms that they register in the seller center input field.

Candidate examples may include:

```text
별모양심지
향명
향조 연상어
```

Do not label candidate terms as final confirmed terms.

If the user says only some terms registered, preserve only those confirmed terms as final.

## Coupang Rule

Coupang search terms may use broader free-input terms.

Rules:

```text
maximum 20 terms
include scent keyword
include candle category
include structure keyword
include material keyword
include use-case keyword
include gift/search intent keyword
```

Do not include unverified claims.
Do not exceed 20 terms.

## Cafe24 Rule

For Cafe24 product fields:

- Do not use the pipe character `|` in product names.
- Do not invent prices.
- Do not invent origin.
- Do not invent manufacturer.
- Author must follow the confirmed project data.
- Product title, browser title, meta keywords, and promotion text must stay consistent.

## SNS Rule

Short-form content rules:

```text
Instagram: 7-12 seconds, one-line text, 3 hashtags
TikTok: 5-9 seconds, minimal text, 2 hashtags
YouTube Shorts: 10-12 seconds, two-line text, 3 hashtags
```

For candle short-form content:

```text
9:16
first 1 second: flame close-up
use crackling sound when available
avoid emotional exaggeration
show structure or burn result
```

## Error Handling

If required input is missing, return only:

```text
입력값 부족 : [부족한 항목]
```

Do not guess.
Do not generate placeholder product facts.
Do not ask multiple follow-up questions.
